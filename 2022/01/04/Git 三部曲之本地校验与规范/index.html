Git 三部曲之本地校验与规范

Posted on 2022-01-04 · 阅读约 2 分钟

引言

加入新公司，我们都会经历的一个转正答辩的必经流程，那么在团队建设与规划这一要点中，我们就可以从 git本地检验与规范 入手，其相对组件库或者脚手架而言，则不需要花费过多精力，并且可以立马在项目中推动起来，达到立竿见影的效果，假如开发一个组件库，可能还得经过多个版本测试验证，才能在正式环境中推广起来。

问题

在创业公司或者是新团队中，其 git 规范往往是不够完善的，几乎都存在以下问题：

  • 历史项目维护与新项目研发并行开发，
  • 历史项目业务往往比较复杂，没人愿意完善相关 git 规范
  • 相关脚手架还未成型，新项目只能复用老项目，老项目又缺少相关规范，导致陷入死循环
  • 新项目中集成 git 规范过于复杂，希望一键集成

当然还有其它问题暂未列出，问题的本质就是没人愿意折腾 git 环境配置，都想着拿来就用，一键配置使用，所以接下来我们就来解决如何让团队成员想用就用，无需过多关心环境配置，解放双手。
